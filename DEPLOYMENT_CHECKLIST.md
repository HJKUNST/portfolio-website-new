# 배포 체크리스트 (Deployment Checklist)

이 문서는 포트폴리오 웹사이트 배포 전/중/후 점검 사항을 정리한 것입니다.

## 1. 배포 전 유의사항

### 1.1 코드 및 환경 점검

#### ✅ 버전 관리 시스템 확인
- [ ] 최신 코드가 Git에 정상적으로 반영되었는지 확인
- [ ] `git status`로 변경사항 확인
- [ ] `git log`로 최근 커밋 내역 확인
- [ ] 배포할 브랜치가 올바른지 확인 (main/master/production)

#### ✅ 환경별 설정 파일 점검
- [ ] `.env.example` 파일이 존재하고 최신 상태인지 확인
- [ ] 운영 환경(PRD)과 개발 환경(DEV) 설정이 분리되었는지 확인
- [ ] 환경 변수가 올바르게 설정되었는지 확인
- [ ] 민감한 정보(API 키, 토큰 등)가 코드에 하드코딩되지 않았는지 확인

#### ✅ 빌드 및 컴파일 확인
- [ ] `npm run build` 성공 여부 확인
- [ ] TypeScript 컴파일 오류 없음 확인
- [ ] ESLint 경고/오류 해결 확인
- [ ] 프로덕션 빌드 크기 확인 (번들 크기 최적화)

### 1.2 테스트 수행

#### ⚠️ 현재 상태: 테스트 파일 없음
- [ ] 단위 테스트(Unit Test) 작성 및 실행
- [ ] 통합 테스트(Integration Test) 작성 및 실행
- [ ] 회귀 테스트(Regression Test) 수행
- [ ] 성능 테스트(Load Test) 수행 (선택사항)
- [ ] 수동 테스트 체크리스트:
  - [ ] 모든 섹션이 정상 렌더링되는지 확인
  - [ ] 스크롤 애니메이션이 정상 작동하는지 확인
  - [ ] 커서 인터랙션이 정상 작동하는지 확인
  - [ ] Dialog 컴포넌트가 정상 작동하는지 확인
  - [ ] 반응형 디자인 확인 (데스크톱 우선)
  - [ ] 접근성 확인 (키보드 네비게이션, 스크린 리더)

#### ✅ CI/CD 파이프라인
- [ ] GitHub Actions 워크플로우 설정 확인 (`.github/workflows/deploy.yml`)
- [ ] 자동 빌드 및 테스트가 통과하는지 확인
- [ ] 배포 자동화 스크립트 검증

### 1.3 롤백(Rollback) 전략

#### ✅ 롤백 방법 준비
- [ ] 이전 버전으로 되돌리는 방법 문서화
- [ ] Vercel/배포 플랫폼에서 롤백 절차 확인
- [ ] Git 태그를 사용한 버전 관리 전략 수립
- [ ] Blue-Green Deployment 또는 Canary Deployment 고려 (선택사항)

#### 롤백 절차
1. 배포 플랫폼(Vercel 등)에서 이전 배포로 롤백
2. 또는 Git에서 이전 커밋으로 되돌리고 재배포
3. 롤백 후 기능 정상 동작 확인

### 1.4 데이터베이스 변경(Migration)

#### ✅ 현재 상태: 데이터베이스 사용 없음
- [ ] 이 프로젝트는 정적 콘텐츠만 사용하므로 데이터베이스 마이그레이션 불필요
- [ ] 향후 데이터베이스 추가 시 마이그레이션 전략 수립 필요

### 1.5 배포 일정 및 영향도 분석

#### ✅ 배포 시간 설정
- [ ] 서비스 이용률이 낮은 시간대 선택 (예: 새벽 2~5시)
- [ ] 배포 일정을 팀과 공유
- [ ] 예상 다운타임 또는 영향 범위 공유

#### ✅ 영향도 분석
- [ ] 배포로 인한 예상 영향 범위 문서화
- [ ] 운영팀, 고객센터, 관련 부서와 사전 공유
- [ ] 배포 중단 공지 필요 여부 확인

## 2. 배포 중 유의사항

### 2.1 무중단 배포(Zero Downtime Deployment)

#### ✅ 배포 방법
- [ ] Vercel은 기본적으로 무중단 배포 지원
- [ ] 배포 중 서비스 정상 작동 여부 모니터링
- [ ] 새 버전 배포 후 이전 버전 유지 (롤백 대비)

### 2.2 로그 및 모니터링 활성화

#### ✅ 로그 확인
- [ ] 배포 중 실시간 로그 확인
- [ ] Vercel 대시보드에서 배포 로그 모니터링
- [ ] 에러 로그 확인 및 대응

#### ⚠️ APM 도구 설정 (선택사항)
- [ ] Vercel Analytics 활성화 고려
- [ ] 외부 모니터링 도구 연동 (예: Sentry, LogRocket)
- [ ] 성능 모니터링 설정

#### ✅ 알림 설정
- [ ] 배포 성공/실패 알림 설정 (이메일, Slack 등)
- [ ] 에러 발생 시 알림 설정

### 2.3 트래픽 부하 관리

#### ✅ 현재 상태: 정적 사이트
- [ ] Vercel은 자동으로 CDN 및 로드 밸런싱 제공
- [ ] 트래픽 급증에 대한 자동 스케일링 지원
- [ ] 배포 후 트래픽 모니터링

### 2.4 배포 프로세스 자동화

#### ✅ CI/CD 설정
- [ ] GitHub Actions를 통한 자동 배포 설정 확인
- [ ] 배포 전 자동 빌드 및 테스트 실행
- [ ] 배포 후 자동 스모크 테스트 실행 (선택사항)

## 3. 배포 후 유의사항

### 3.1 기능 정상 동작 확인

#### ✅ Smoke Test 체크리스트
- [ ] 홈페이지 로딩 확인
- [ ] 모든 섹션 렌더링 확인:
  - [ ] Hero Section
  - [ ] Values Arrow Field Section
  - [ ] Shared Craftery Path Section
  - [ ] Teams Section
  - [ ] Looking For Team Section
- [ ] 스크롤 애니메이션 동작 확인
- [ ] 커서 인터랙션 동작 확인
- [ ] Dialog 모달 동작 확인
- [ ] 링크 및 버튼 동작 확인

#### ✅ 성능 확인
- [ ] 페이지 로딩 속도 확인 (Lighthouse 사용 권장)
- [ ] 이미지 최적화 확인
- [ ] 번들 크기 확인
- [ ] Core Web Vitals 점수 확인

### 3.2 장애 대응 및 롤백 절차 실행 가능 여부 확인

#### ✅ 롤백 준비
- [ ] 롤백 절차 문서화 확인
- [ ] 이전 버전 접근 가능 여부 확인
- [ ] 긴급 패치(Hotfix) 프로세스 준비

### 3.3 사용자 피드백 반영

#### ✅ 피드백 수집 체계
- [ ] 사용자 피드백 수집 방법 확인
- [ ] 버그 리포트 처리 프로세스 수립
- [ ] UI/UX 개선 요청 기록 및 추적

### 3.4 성능 및 보안 점검

#### ✅ 성능 점검
- [ ] Lighthouse 성능 점수 확인
- [ ] 페이지 로딩 속도 측정
- [ ] 이미지 최적화 확인
- [ ] 번들 크기 최적화 확인

#### ✅ 보안 점검
- [ ] OWASP 보안 가이드라인 준수 확인
- [ ] HTTPS 적용 확인
- [ ] CORS 설정 확인 (필요시)
- [ ] 환경 변수 보안 확인
- [ ] 의존성 취약점 스캔 (`npm audit`)
- [ ] Content Security Policy (CSP) 설정 확인

## 4. 배포 플랫폼별 체크리스트

### Vercel 배포 시

- [ ] Vercel 프로젝트 연결 확인
- [ ] 환경 변수 설정 확인 (Vercel 대시보드)
- [ ] 도메인 설정 확인
- [ ] 빌드 설정 확인 (`next.config.ts`)
- [ ] 배포 후 프리뷰 URL 확인
- [ ] 프로덕션 도메인 확인

### 기타 플랫폼 배포 시

- [ ] 플랫폼별 설정 파일 확인
- [ ] 빌드 명령어 확인
- [ ] 시작 명령어 확인
- [ ] 포트 설정 확인
- [ ] 환경 변수 설정 확인

## 5. 체크리스트 실행 방법

1. 배포 전: 이 문서의 "1. 배포 전 유의사항" 섹션을 순서대로 확인
2. 배포 중: "2. 배포 중 유의사항" 섹션을 실시간으로 모니터링
3. 배포 후: "3. 배포 후 유의사항" 섹션을 순서대로 확인

각 항목을 체크한 후 배포를 진행하세요.
